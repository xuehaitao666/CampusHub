《AI 协作契约》模板

团队名称：倒地爆冲
日期：2026/4/13

第一条：AI 使用原则
- AI 是工具，工程师是责任主体
- 所有 AI 输出必须经过人工审查后才能合入项目

第二条：AI 使用范围
- 允许使用 AI 的场景：生成基础模板代码（如 Vue 组件骨架、SQL 表结构初稿）、配置 CI/CD 流水线与 .gitignore 文件、生成测试用例初稿、排查报错信息、辅助优化 UI 样式与动画特效。
- 禁止直接使用 AI 输出（需额外审查）的场景：核心业务逻辑的实现（如订单流转状态机、智能匹配算法）、涉及数据库增删改查的关键接口逻辑、复杂的权限校验与 JWT 鉴权机制。此类代码必须完全理解，并在队友 Review 后方可合入。
- 完全禁止使用 AI 的场景：捏造 P1 阶段的用户调研数据（问卷与访谈记录）、替写个人的《AI 协作反思日志》的核心反思部分、在未经修改的情况下直接上传包含系统环境变量或数据库真实密码的 AI 示例代码。

第三条：过程记录要求
- 所有与 AI 的交互 Prompt 必须保留日志
- Git commit message 必须标注 [AI-assisted] 或 [Human-written]
- 每阶段填写《AI 协作反思日志》

第四条：代码合入规则
- AI 生成的代码必须通过至少一名团队成员的 Code Review
- 每位成员每个编码阶段至少有 2 个完全手写的核心函数

第五条：违约处理
- 团队内部约定的违约处理方式：
  1. 首次违约（如：Commit 漏写标签、未 Review 直接合入 AI 代码）：需在团队群内检讨，并负责撰写本周的团队进度周报。
  2. 再次违约或导致严重 Bug：必须手动重写该部分问题代码，并请全队喝奶茶/饮料。
  3. 严重违约（如：代码完全依赖 AI 且无法向队友解释其运行逻辑）：撤销该部分代码的贡献度，由该成员独立承担修复工作。
