# 核心用例详细描述

## 用例1：发布跑腿任务

**用例名称**：发布跑腿任务  
**参与者**：学生用户（发布者）

### 前置条件
- 用户已注册账号
- 用户已完成学号认证
- 用户处于登录状态

### 基本流程
1. 用户进入“发布任务”页面  
2. 选择任务类型（快递/外卖/排队/搬运）  
3. 填写任务信息（时间、地点、价格、描述）  
4. 点击“发布”按钮  
5. 系统校验信息完整性  
6. 系统发布任务并展示在任务列表中  

### 替代流程
- **A1：信息填写不完整**  
  → 系统提示“请完善任务信息”，返回填写页面  

- **A2：价格不合理（过低或过高）**  
  → 系统提示价格建议范围  

### 后置条件
- 任务成功发布  
- 其他用户可以浏览并接单

## 用例2：接单并完成任务

**用例名称**：接单并完成任务  
**参与者**：学生用户（接单者）、学生用户（发布者）

### 前置条件
- 平台存在可接任务  
- 任务未被接单  
- 发布者已支付担保资金  

### 基本流程
1. 接单者浏览任务列表  
2. 选择任务并点击“接单”  
3. 发布者收到通知并确认接单  
4. 接单者执行任务  
5. 接单者上传完成凭证（图片/说明）  
6. 发布者确认任务完成  
7. 系统将资金支付给接单者  

### 替代流程
- **B1：发布者拒绝接单请求**  
  → 接单失败，用户可选择其他任务  

- **B2：接单者未按时完成任务**  
  → 发布者可取消任务或发起纠纷  

- **B3：凭证不符合要求**  
  → 发布者拒绝确认，要求重新提交  

### 后置条件
- 订单完成  
- 双方可进行评价  
- 信用分更新

## 用例3：担保交易与纠纷处理

**用例名称**：担保交易与纠纷处理  
**参与者**：学生用户、管理员、平台系统

### 前置条件
- 订单已创建  
- 发布者已完成支付（资金进入平台托管）  

### 基本流程
1. 发布者支付任务费用至平台  
2. 平台冻结资金（托管状态）  
3. 接单者完成任务并提交凭证  
4. 发布者确认完成  
5. 平台将资金转给接单者  

### 替代流程
- **C1：发布者拒绝确认完成**  
  → 接单者可发起申诉  

- **C2：接单者未履约**  
  → 发布者申请退款  

- **C3：发生纠纷**  
  → 管理员介入  
  → 查看聊天记录与凭证  
  → 判定责任并分配资金  

### 后置条件
- 交易完成或退款  
- 纠纷记录存档  
- 用户信用评分更新
