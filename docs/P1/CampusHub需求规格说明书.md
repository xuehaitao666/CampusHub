# 校园互助服务平台 CampusHub
## 软件需求规格说明书（SRS）
**版本**：V1.0  
**日期**：2026-04-19  
**目标用户**：在校大学生

---

## 1. 引言
### 1.1 目的
本文档为校园互助服务平台（CampusHub）的软件需求规格说明书，定义平台的功能、性能、安全与约束要求，作为后续设计、开发、测试与验收的依据。

### 1.2 范围
- **产品名称**：CampusHub 校园互助服务平台
- **目标场景**：快递代取、学习辅导、二手交易、组队匹配
- **运行环境**：微信小程序、H5网页端
- **排除范围**：不提供代写、代考、代课等违规服务；不支持跨校服务

### 1.3 定义与缩写
- SRS：软件需求规格说明书
- 发布者：发布互助需求的用户
- 接单方：承接互助需求的用户
- 担保交易：平台托管资金，任务完成后结算
- 校内认证：学号/校园统一认证完成学生身份核验

### 1.4 参考文献
1. IEEE 830-1998 标准
2. 校园互助平台用户问卷调研结果（N=50）
3. 在校大学生用户访谈记录（N=8）

## 2. 总体描述
### 2.1 产品视角
面向在校大学生的校内可信互助平台，通过身份认证、担保交易、信用评价解决匹配难、信任低、交易无保障问题。

### 2.2 用户特征
- 在校大学生，18–25岁
- 高频需求：代取、辅导、二手、组队
- 重视安全、隐私、免费、简洁
- 依赖微信，厌恶信息泄露

### 2.3 运行环境
- 前端：微信小程序
- 后端：云服务器
- 第三方：微信登录、校园认证、支付、短信接口

### 2.4 设计约束
1. 仅对校内认证学生开放
2. 有偿任务强制担保交易
3. 学号、手机号不对外展示
4. 禁止违规需求与交易
5. 基础功能永久免费

### 2.5 假设与依赖
- 可完成校园身份核验
- 微信、支付接口可用
- 用户接受信用与仲裁规则
