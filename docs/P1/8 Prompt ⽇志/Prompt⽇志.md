# Prompt 日志（Prompt Log）— Phase 1 需求分析

## 1. Prompt 使用记录

### Prompt 1：需求规格说明书生成

**Prompt：**  
请为一个校园互助服务平台生成完整的需求规格说明书，平台支持学生发布和接单各类互助需求（快递代取、学习辅导、二手交易、组队匹配等），目标用户为大学生。请按IEEE 830标准格式输出。

**输出评价：**  
- 结构完整（符合SRS格式）  
- 内容偏通用，缺乏校园场景细节  
- 存在部分过度设计  

**改进点：**  
需加入“基于调研结果”约束  

---


### Prompt 2：需求对比分析
**任务阶段**：AI需求盲区分析  

**Prompt：**  
请对比AI生成的需求与人工调研结果，从覆盖、遗漏、捏造和歧义四个方面进行分析，并以表格形式输出。

**输出评价：**  
- 分析结构清晰  
- 结论较合理  

**改进点：**  
需结合具体调研数据增强说服力  

---

### Prompt 3：调研驱动需求重构
**任务阶段**：需求修正  

**Prompt：**  
请基于以下用户调研结果重新生成需求规格说明书，突出高频需求（跑腿、问答）、身份认证、担保交易等关键功能，并避免过度设计。

**输出评价：**  
- 更贴近实际需求  
- 优先级更加清晰  

**改进点：**  
需进一步细化功能描述  

---

### Prompt 4：用例建模
**任务阶段**：任务3 用例图  

**Prompt：**  
为校园互助平台设计用例模型，要求至少包含3类角色和15个用例，并覆盖核心业务流程。

**输出评价：**  
- 满足数量要求  
- 覆盖核心功能  

**改进点：**  
需补充异常流程  

---

### Prompt 5：详细用例描述
**任务阶段**：核心用例编写  

**Prompt：**  
请为以下功能编写详细用例描述：发布任务、接单完成、担保交易。要求包含前置条件、基本流程、替代流程和后置条件。

**输出评价：**  
- 结构规范  
- 初始版本缺少部分异常流  

**改进点：**  
补充违约、拒绝、纠纷处理流程  

---

### Prompt 6：用户故事生成
**任务阶段**：用户故事编写  

**Prompt：**  
请为校园互助平台生成15条用户故事，采用“作为…我希望…以便…”格式，并提供验收标准。

**输出评价：**  
- 格式正确  
- 验收标准基本可测试  

**改进点：**  
增加优先级划分  

---

### Prompt 7：用户故事优先级划分
**任务阶段**：需求优先级分析  

**Prompt：**  
基于需求编写至少15条用户故事，采用“作为…我希望…以便…”格式，并附带可测试的验收标准。

**输出评价：**  
- 分类清晰  
- 符合实际业务逻辑  

**改进点：**  
需结合调研数据说明优先级依据  

---

## 2. Prompt 演进总结

### 初始问题
- Prompt过于宽泛  
- 缺乏约束条件  
- 未提供真实数据  

### 改进策略
- 加入“基于调研”的明确约束  
- 明确输出格式（IEEE 830 / 用例结构）  
- 分阶段使用Prompt（逐步生成）  
- 限制范围（避免过度设计）  

### 最终效果
- AI输出从“通用模板” → “贴近真实需求”  
- 减少无效内容，提高可用性  
- 提升文档规范性与完整性  

---

## 3. 总结

本阶段中，Prompt设计对AI输出质量影响显著。  
通过逐步优化Prompt（增加约束、提供上下文、细化任务），成功提升了AI生成内容的准确性与可落地性。
